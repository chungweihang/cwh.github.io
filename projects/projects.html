Facial Recognition and Detection

Spring, 2007 (330 lines)

Abstract We demonstrate the ability of combining facial detection techniques with a LEGO MINDSTORMS NXT robots using Commercial Off The Shelf (COTS) components, and show the feasibility of combining this facial recognition schemes, that given proper integration may allow such ``toys'' to identify people in realtime. We achieved realtime facial detection with between 5 and 9 fps, and discovered that while not extremely accurate, facial recognition may be applied to returned images to ``sort-of'' recognize individuals. (proposal | report)

Remote Access System

Fall, 2003 (518 lines)

Abstract We implemented a rsh-like system, called remote access system, where users can run problems, use pipe ("|") and redirect (">"), set environment variables, and so on. For example, users can run the following commands. (report) 

% cat test.html > test1.txt
% removetag test.html |2          # This pipe will pipe stdout to next next command. 
% removetag test.html |1          # This pipe will pipe stdout to next command. (Note: merge with the previous one)
% number                          # The command's stdin is from the previous pipe.
  1 
  2 Test 
  3 This is a test program
  4 for ras.
  5 
  6 
  7 Test 
  8 This is a test program
  9 for ras.

Romote Working Ground

Fall, 2003 (2036 lines)

Abstract Extending the remote access system, we added chat-like functions using two concurrent paradigm: 1) single-process concurrent paradigm, and 2) concurrent connection-oriented paradigm (semaphore, shared memory). We implemented who, name, yell (broadcast), tell, list, redirect (">"), and receive ("<") functions. (report) 

# client 1 
% ls -l | 
% cat >2              # redirect the stdout to client 2

# client 2 
% number <1 >1        # receive from client 1 and redirect back immediately

Batched Remote Access System in HTTP

Fall, 2003 (1958 lines)

Abstract We implemented a simple HTTP server on both Unix and Windows (winsock), which can show HTML pages, images, and execute CGI. The CGI takes remote access systems and batch files as parameters, connects to those remote access systems, executes the batch files, outputs results as text webpages.
